
# 데이터 레이크, 데이터 웨어하우스, 데이터 마트 개념 및 다중 클라우드 구성 예시

## 1. 데이터 레이크 (Data Lake)

### 개념
- 원시 형태의 정형, 반정형, 비정형 데이터를 대용량 저장
- 스키마를 미리 정의하지 않고 데이터 가공 전 그대로 저장
- 다양한 분석 및 AI/ML 활용 목적

### 서비스 및 기술

| 클라우드 | 서비스/기술                   | 역할 및 설명                          |
|---------|-----------------------------|------------------------------------|
| GCP     | Cloud Storage, BigLake Metastore | 원시 데이터 저장, 메타데이터 관리         |
| AWS     | Amazon S3, AWS Lake Formation     | 대규모 객체 저장소, 데이터 카탈로그 관리  |
| Azure   | Azure Data Lake Storage Gen2        | 확장성 높은 데이터 저장 및 관리           |

***

## 2. 데이터 웨어하우스 (Data Warehouse)

### 개념
- 여러 소스의 데이터를 정제·구조화해 분석에 최적화된 저장소
- 전사적 데이터 통합, 리포팅과 경영분석 지원

### 서비스 및 기술

| 클라우드 | 서비스/기술                   | 역할 및 설명                          |
|---------|-----------------------------|------------------------------------|
| GCP     | BigQuery                     | 완전관리형 클라우드 데이터 웨어하우스       |
| AWS     | Amazon Redshift              | 확장성 높은 클라우드 DW 솔루션              |
| Azure   | Azure Synapse Analytics      | 엔드투엔드 분석 플랫폼, DW 및 빅데이터 통합 서비스 |

***

## 3. 데이터 마트 (Data Mart)

### 개념
- 웨어하우스에서 특정 부서·주제별 필요한 데이터만 추출·가공해 별도 저장
- 신속한 데이터 접근과 분석 지원

### 서비스 및 기술

| 클라우드 | 서비스/기술                   | 역할 및 설명                          |
|---------|-----------------------------|------------------------------------|
| GCP     | BigQuery (뷰, 별도 테이블), Looker, Data Studio | 부서·상품별 뷰 생성, BI 도구 연동        |
| AWS     | Amazon Redshift 데이터베이스, QuickSight                | 분산 쿼리 및 시각화 BI 도구 연동          |
| Azure   | Azure Synapse 데이터베이스, Power BI                    | 부서별 마트 구성, 대시보드 및 리포팅 도구  |

***

## 기타 용어 및 개념 정리

### 1. 데이터 레이크하우스 (Data Lakehouse)
- 데이터 레이크의 유연성과 데이터 웨어하우스의 구조화된 분석 기능을 결합한 최신 아키텍처
- 원시 데이터 저장과 고성능 분석이 하나의 플랫폼에서 가능
- 예: Databricks, Delta Lake, Apache Hudi, Apache Iceberg 등이 대표적

### 2. ETL / ELT
- 데이터 추출(Extract), 변환(Transform), 적재(Load)의 약자(ETL), 변환과 적재의 순서를 바꾼 ELT
- 데이터 레이크에서 웨어하우스로 데이터 이동 시 필수적인 데이터 파이프라인 처리 과정
- 주요 도구: Apache NiFi, AWS Glue, Google Cloud Dataflow, Azure Data Factory 등

### 3. 데이터 카탈로그 (Data Catalog)
- 데이터 자산의 메타데이터를 관리하고 검색, 거버넌스를 지원하는 시스템
- 데이터 위치, 구조, 품질, 소유자를 추적해 데이터 활용 효율성과 규정 준수 강화
- 예: AWS Glue Data Catalog, Azure Purview, Google Cloud Data Catalog

### 4. 스트리밍 데이터 처리
- 실시간으로 생성되는 데이터(예: IoT 센서, 클릭스트림)를 빠르게 처리하는 기술
- Apache Kafka, AWS Kinesis, Google Cloud Pub/Sub, Azure Event Hubs 등이 대표적

### 5. 분산 처리 프레임워크
- 대용량 데이터를 분산 저장 및 처리하는 기술
- Apache Hadoop, Apache Spark, Google Dataflow 등

### 6. 빅데이터 분석 도구
- 데이터 분석과 시각화 도구로, 대화형 분석 및 리포팅에 사용
- Tableau, Power BI, Looker, AWS QuickSight 등
